Morning · 9 am – 1 pm

01
Understand how agents actually work 2 hrs
  • Read the ReAct paper (Reason + Act loop) — this is the mental model you'll implement
  • Study the agent loop: observe → think → act → observe (repeat until done)
  • Learn what a "tool" is in agent terms — a function the model can call with structured args
  • Understand context window management: why agents forget and how to work around it
ReadReAct: Synergizing Reasoning and Acting in Language Models (arxiv.org/abs/2210.03629)
ReadAnthropic's "Building effective agents" guide (docs.anthropic.com)
WatchAndrej Karpathy — "Intro to LLMs" (covers tool calling and agentic loops at ~1hr mark)
02
Design the model-agnostic interface 2 hrs
  • Study how OpenAI, Anthropic, and Gemini APIs differ — message formats, tool calling schemas, streaming
  • Define a ModelProvider abstract class with a single complete(messages, tools) method
  • Map each provider's quirks into adapter classes that conform to your interface
  • Decide your tool schema format — OpenAI's JSON Schema convention is the de facto standard

Afternoon · 2 pm – 7 pm

03
Build the provider adapters 2.5 hrs
  • Implement OpenAIProvider — wraps gpt-4o, handles tool_calls in response
  • Implement AnthropicProvider — wraps Claude, maps tool_use blocks
  • Implement OllamaProvider — local models, no API key needed (great for testing)
  • Write a unified Message dataclass that every adapter outputs
  • Add streaming support behind a flag — you'll need it for UX later
04
Build the core agent loop 2.5 hrs
  • Create an Agent class that takes a provider, a list of tools, and a system prompt
  • Implement the run(user_input) loop: call model → parse tool calls → execute → feed result back
  • Add a max_iterations guard — prevents infinite loops
  • Implement memory: simple list-based conversation history passed with every call
  • Handle parallel tool calls if the model requests them

Morning · 9 am – 1 pm

Build a useful tool suite 2 hrs
  • web_search(query) — scrape DuckDuckGo or use SerpAPI free tier
  • read_file(path) and write_file(path, content) — file I/O
  • run_python(code) — execute code in a subprocess sandbox
  • http_get(url) — fetch and return page content
  • Use a @tool decorator that auto-generates the JSON schema from type hints and docstrings

  • Add a remember(fact) and recall(query) tool the agent can call explicitly
  • Plan-then-execute mode: first call produces a step-by-step plan, subsequent calls execute each step

Key design decisions

?
Python vs TypeScript
  • Python: better ecosystem (LangChain, ChromaDB, numpy), easier scripting, more agent examples online
  • TypeScript: better for web integration, stronger types catch bugs early, Vercel AI SDK is excellent
  • Recommendation: Python for the weekend build — you'll move faster. TypeScript if the agent runs in a web app
?
Build from scratch vs use a framework
  • Build from scratch (this plan): you learn how every part works, easier to debug and extend
  • LangChain/LangGraph: powerful but heavy — you'll fight abstractions until you understand the fundamentals
  • Recommendation: build scratch first. After the weekend, look at LangGraph for production orchestration
